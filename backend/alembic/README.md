Alembic setup for the project.

To run migrations locally:

1. Install alembic in your backend virtualenv:

```bash
pip install alembic
```

2. From the `backend` folder, initialize env and run upgrade:

```bash
cd backend
alembic upgrade head
```

3. To create new revisions in the future:

```bash
alembic revision -m "describe change" --autogenerate
```

Notes:
- `alembic.ini` is configured to use the local SQLite DB `sqlite:///./employee.db`.
- `env.py` imports `Base.metadata` from `app.config.database`, so autogenerated revisions will pick up model changes.
